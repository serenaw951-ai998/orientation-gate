# 心旋 · Product Direction Update
> Human State Reflection System · v0.1
> SenuxTech · 2026

---

## What This Is Now

This project is no longer just a music prompt generator.

The direction is shifting toward a **lightweight Human State Reflection System**.

Music is no longer the core product. Music becomes an emotional expression layer after reflection.

---

## What The System Should Do

- Help users notice their emotional state
- Reflect on subtle internal changes
- Observe emotional movement over time
- Gently translate feelings into music
- Help users feel seen without being diagnosed

---

## What This Is NOT

- A therapy tool
- A psychological diagnosis system
- A personality ranking system
- A spirituality app
- An emotional scoring system

---

## What To Avoid

- Clinical language
- Mental health diagnosis
- Spiritual / quantum wording
- Exaggerated emotional claims
- "AI knows you" framing

---

## Tone

- Calm
- Observant
- Reflective
- Human
- Emotionally intelligent
- Non-judgmental

The experience should feel closer to:
> **"a mirror for your current state"**

Rather than:
> "an AI that analyzes your personality."

---

## Core Product Structure

### 1. State Input Layer

Focus: *"Where are you emotionally right now?"*

Keep:
- Short scene selections
- Free-text feeling input
- Emotional sliders

Add — **Trajectory Intention**:
- I want to slow down
- I want to feel understood
- I want to release tension
- I want to reconnect
- I want to focus again

---

### 2. Reflection Layer *(NEW CORE)*

This becomes the most important part of the product.

Before generating music, generate:
- Emotional reflection
- Subtle state description
- Gentle observation
- Emotional movement language

The tone should NOT:
- Diagnose
- Explain the user psychologically
- Tell users what is wrong

Instead:
- Reflect
- Describe
- Notice
- Gently articulate

**Example output:**
> "You may not be trying to solve everything right now. You may simply need a little more space to breathe."

---

### 3. Trajectory Layer *(NEW)*

A lightweight emotional trajectory system.

Instead of showing only the current state, show movement over time.

Use `localStorage` only for now.

Examples:
- Compared to previous entries
- Tension slightly decreased
- Expression increased
- Recovery still low

This should feel like: **"state awareness"**
NOT: "emotional scoring."

---

### 4. Music Layer

Music appears **AFTER** reflection.

Music is now:
- An emotional translation layer
- A resonance layer
- An expression pathway

Reduce the visual dominance of prompt generation.

**The order becomes:**
```
State → Reflection → Trajectory → Music
```
NOT: Prompt generation first.

---

### 5. Save This Moment

Save:
- Current state
- Reflection text
- Trajectory snapshot
- Music direction
- Timestamp

Goal: build emotional memory and state continuity.

---
